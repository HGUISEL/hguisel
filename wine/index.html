WINE

WINE (Warning mINEr) is a mining tool that can identify complex and unique code examples from the reports generated by a static bug finder automatically for tool designers to find wrong and RW-prone rules.

RW (Rejected Warning) is rejected to be fix after being checked by users although warnings are correct in terms of the rule's original intention.

WW (Wrong Warning) is caused by incorrect implementation of the rules, i.e., bugs in the rule implementation caused by designers.

TW (True Warning) is useful warning that can be helpful to the users to find and correct the bugs in the program.

There are two assumptions.
1) Unique and low frequent patterns of code in a detection report are unusual, which may be a bug (TW), and also, common and high frequent patterns of code in a detection report are normal because they may be ignored cases, which may not be a bug (RW).
2) Complex patterns of examined code need more complex rule design and are easy to be a missed case of design. The complex code examples may indicate WW generated by bugs in the implementation of rules.

Reported PMD issues

* One issue contains at least one case. It means that there are at least one different case to occur problem in implementation or design in the bug finder. There would be one or more cases in one issue.

* Issue Status: O (Open), F (Fixed), C (Closed), AR (Already Reported by others)

* Type: WW (Wrong Warning), NR (New Rule), DE (Documentation Error), RR (Rule Refactoring)
